## 技术方案

> 详细方案见 `evidence/claude/technical-plan-0.0.1.json`，本节为摘要。

### 架构决策

单 FastAPI 服务器同时承载 REST API 和编译后的 React SPA 静态资源。前端通过 React Router 实现客户端路由，后端通过中间件统一处理 CORS、缓存头、认证和静态资源版本令牌。数据层 SQLAlchemy ORM + SQLite 文件数据库，附件存储本地文件系统。

### 前端选型

**React 18 + Vite 5 + TypeScript 5 + Ant Design 5**

选择理由：
1. Vite `base` 配置原生支持 `/projects/contract-hub/` 子路径，无需后端 URL 重写
2. React Router v6 `basename` 与 Vite base 一致，客户端路由自动带前缀
3. Vite 构建注入 `?v=0.0.1` 版本令牌，配合后端 Cache-Control 头实现缓存策略
4. TypeScript 类型系统与 Pydantic 后端模型对应，减少接口缺陷
5. Ant Design 5 成熟后台管理组件（Table、Form、Upload、Modal），加速 MVP 开发

备选方案：Vue 3（生态略小）、Vanilla JS（维护成本高）。

### 数据模型

- **users**：id, username(unique), password_hash(bcrypt), role(admin|user), created_at
- **contracts**：id, title, content, status(draft→pending_review→active→expired→terminated), creator_id(FK), created_at, updated_at
- **attachments**：id, filename, original_filename, file_path, file_type(pdf|doc|docx), file_size, contract_id(FK), uploader_id(FK), uploaded_at

### API 设计

前缀 `/projects/contract-hub/api`，JWT Bearer 认证，20 个端点覆盖认证、用户 CRUD、合同 CRUD+状态流转、附件上传下载。

### 缓存策略

- HTML：FastAPI 中间件设置 `Cache-Control: no-cache` 真实 HTTP 响应头
- 静态资源 JS/CSS：Vite 构建注入 `?v=0.0.1` 令牌 + 长缓存
- 禁止使用 `<meta http-equiv>` 标签（浏览器忽略其缓存语义）

### 主要风险

1. **路由一致性**：Vite base / React Router basename / FastAPI mount_path 三者必须一致性验证
2. **缓存头验收**：`curl -I` 验证 Cache-Control 真实响应头
3. **文件安全**：魔数检测防扩展名伪装
4. **SQLite 并发**：WAL 模式缓解，MVP 阶段可接受
5. **Token 安全**：内存存储，后续迭代考虑 httpOnly cookie
